## Untuk siapa halaman ini

- Owner atau admin bisnis yang baru pertama kali membuka Nizam
- Tim implementasi yang perlu memahami urutan setup
- Developer yang ingin tahu dokumen mana yang perlu dibaca dulu

## Urutan baca tercepat

1. **Peta Produk** untuk memahami ruang lingkup modul.
2. **Konsep Inti** untuk organisasi, Cabang, approval, dan integrasi lintas modul.
3. **Referensi Teknis** jika Anda bekerja pada implementasi, integrasi, atau debugging.

## Catatan implementasi

Platform docs ini dibangun sebagai project terpisah di folder `/docs` agar mudah di-host di Railway dengan root directory yang sama. Dokumen referensi lama tetap dipertahankan dan disinkronkan otomatis ke section **Reference**.
