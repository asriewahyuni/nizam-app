## Isi docs ini

- **Product & operations** untuk owner, admin, finance, inventory, sales, dan tim operasional.
- **Developer & integration** untuk arsitektur, Open API, webhook, dan pola modul.
- **Reference library** yang mengambil file Markdown sumber dari folder `/docs` lama dan menyajikannya ulang di platform baru.

## Prinsip utama

- Gunakan istilah **Cabang** secara konsisten.
- Perlakukan Nizam sebagai ERP yang **terhubung lintas modul**, bukan kumpulan fitur silo.
- Gunakan docs ini sebagai **source of truth yang ramah baca**, sementara file Markdown sumber tetap dipertahankan untuk jejak historis dan drafting internal.
