# 2026-05-12

## Roles & Permissions Fix (Akses & Jabatan)

### Problem
LMS tidak muncul di halaman Akses & Jabatan (`/settings/roles`). Root cause: `MODULE_CATEGORIES` di roles page adalah hardcoded stale list yang tidak sinkron dengan `module-registry.ts`.

### Fix Round 1 — Sync hardcoded list dengan registry
- Renamed `learning` (Peningkatan Kompetensi) → `lms` (LMS / Lembaga Pelatihan)
- Added missing modul: Warehouse (WMS), Workshop & Service, Job Order (Jasa), Proyek & Konstruksi, Landing Penjualan (Sales Page)
- Moved LMS from HRIS → OPERASIONAL (sesuai definisi Agan: LMS = business type = modul operasional)

### Fix Round 2 — Filter by org activation status
- Added `getActiveOrgEnabledModules()` server action di `roles.actions.ts` — returns `enabledModules` array from `getActiveOrg()`
- Added `GATED_MODULE_MAP` — maps role module IDs to registry keys for business type & add-on modules
- `activeCategories` now filters out gated modules not in `enabledModules`
- Pillar sub-modules (Finance, HRIS, etc.) always visible since pillars are always active
- Empty categories auto-hidden

### Decisions
- Agan confirmed: "modul tidak aktif, tidak perlu muncul" → roles page must dynamically filter
- Gated modules: factory (Manufacturing), workshop (Workshop), joborder (Job Order), project (Project & Construction), fleet (Fleet & Rental), lms (LMS), pos (POS), salespage (Sales Page)

### Future concern (noted for Agan)
- `MODULE_CATEGORIES` is still hardcoded. Each new module added to registry requires manual edit here.
- Ideal fix: generate categories dynamically from `module-registry.ts` with auto-permission mapping.
