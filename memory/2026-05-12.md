## Auto FX Gain/Loss (Phase 2.4) — DONE ✅
### Backend (`modules/accounting/actions/forex.actions.ts`)
- `recordFxGainLoss(referenceType, referenceId, settlementRate?)` — fungsi utama
  1. Get original transaction (sale/purchase) → currency_code, exchange_rate, grand_total
  2. Get current exchange rate
  3. Calculate: `amountInForeign = grandTotal / rateAtTransaction` → `fxDiff = (amountInForeign * rateAtSettlement) - grandTotal`
  4. Find or create CoA accounts: 'Laba Selisih Kurs' (4-8010), 'Rugi Selisih Kurs' (6-8010)
  5. Counterpart account: piutang dagang (1-1010) for sales, hutang dagang (2-1010) for purchases
  6. Create journal entry: Debit/Credit counterpart + gain/loss account
  7. Record in `forex_realized_gl` table (already existed from migration 1304)
- `getFxGainLossHistory(orgId)` — riwayat realisasi
- `deleteFxGainLoss(entryId)` — hapus record + journal entry terkait

### Frontend (`app/(dashboard)/accounting/forex/`)
- **ForexClient.tsx**: Summary cards (total gain, total loss, net), history table, EmptyState, delete
- **page.tsx**: Server component wrapper

### Integration
- **Purchasing**: `createPurchasePayment` → calls `recordFxGainLoss('PURCHASE', id)` after payment processed (non-blocking)
- **Sales**: `processSalesPayment` → calls `recordFxGainLoss('SALE', id)` after payment processed (non-blocking)
