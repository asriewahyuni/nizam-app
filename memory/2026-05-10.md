## Bug Fixes (Execution)

### 1. Payroll JE Empty Lines - FIXED ✅
**Root cause:** `process_payroll_payment` RPC's loop over `payslip_lines` used `AND account_id IS NOT NULL` filter. When CoA wasn't installed yet, `generate_payslips_for_run` couldn't find a valid account → `account_id` was NULL → loop produced 0 rows → JE header created but zero lines.

**Fix:**
- `process_payroll_payment`: Added `FALLBACK` block — if loop produces no lines AND `v_total_net > 0`, auto-find expense account (6001) and create 1 line
- `generate_payslips_for_run`: Multi-level account fallback: 6001 → any 6xxx → any org account (never leave account_id NULL)
- Both RPCs updated directly on Railway PostgreSQL

### 2. NIK Auto-Counter Resets Per Branch - FIXED ✅
**Root cause:** `getNextNik()` used `employees` state which only contains branch-filtered employees. Switching branches → counter resets.

**Fix:**
- Added `allEmployees` prop to `HrisClient` — fetched server-side without branch filter
