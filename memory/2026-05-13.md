## Experience Reflection

### Cognitive Framework: Remediasi Hardcoded Data vs Sumbernya
When a user reports "X tidak muncul di Y padahal sudah diaktifkan", the pattern is:
1. Check if Y reads from a static/hardcoded list instead of the dynamic source
2. Sync the static list with the source, OR make Y dynamic
3. For roles/permissions: the `MODULE_CATEGORIES` in the roles page was stale vs `module-registry.ts`. Quick fix = sync manually. Proper fix = generate from registry dynamically.

### Lessons: Permission/access patterns
- Permission IDs in MODULE_CATEGORIES (`lms:read`, `warehouse:write`) are just labels for UI checkboxes. Backend enforcement uses a separate RBAC mechanism (`roles.permissions` array). So renaming permission IDs doesn't break functionality as long as the IDs are consistent within the UI.
- Gated modules (business type & add-on) need to be filtered by `enabledModules` from org data to avoid showing permissions for inactive modules.
- The `activeRoleDepartmentIds` filter (from role's `department_ids`) is for showing/hiding entire CATEGORIES. The `enabledModules` filter is for showing/hiding individual MODULES within a category.

### Cognitive Framework: Multi-currency Implementation Pattern
Adding multi-currency to a new module (sales → purchases) follows the same pattern:
1. Add `currency_code`, `exchange_rate` to the `Create*Data` type/interface
2. Include these fields in the DB insert (both DRAFT and PUBLISH flows)
3. Add `CurrencyPicker` to the form header
4. Add `CurrencyBadge` to list items (only for non-IDR)
5. Reset currency state in form reset function
6. Server action must handle currency sync (RPC flows need separate update call)

### Pitfalls Avoided
- Don't wrap server-only functions (`getActiveOrg` → `cookies()`) in client components. Use server actions instead.
- The `process_purchase_atomic` RPC doesn't handle currency fields → need separate `updatePurchaseRecord` call after RPC for currency sync
- Module registry keys use PascalCase with spaces (`'Fleet & Rental'`, `'Project & Construction'`) but GATED_MODULE_MAP keys are lowercase IDs. Normalization needed for comparison (lowercase + remove whitespace)
